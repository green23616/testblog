## Issues Encountered

### Issue 1: Missing Autoprefixer
**Problem**: Build failed with "Cannot find module 'autoprefixer'" error
**Cause**: Autoprefixer was not included in initial dependencies
**Solution**: Added `autoprefixer@10.4.22` as dev dependency
**Prevention**: Always verify PostCSS plugin dependencies when using Tailwind

### Issue 2: Undefined Tailwind Classes
**Problem**: Build failed with "border-border class does not exist" error
**Cause**: Used custom Shadcn/ui classes without defining CSS variables
**Solution**: Simplified `globals.css` to basic Tailwind directives
**Note**: Will add custom design tokens when setting up Shadcn/ui components in Phase 2

### Issue 3: Interactive CLI Prompts
**Problem**: `pnpm create next-app` required interactive input
**Cause**: Command waited for user responses about React Compiler
**Solution**: Manually created `package.json` with specific versions and ran `pnpm install`
**Benefit**: More control over exact dependency versions

## Significant Decisions

### 1. React Version
**Decision**: Use React 19.2.0 (upgraded from 19.0.0)
**Rationale**: Latest stable version with improved performance and features
**Impact**: May require compatibility checks for certain libraries

### 2. Manual Project Setup
**Decision**: Create package.json manually instead of using CLI wizard
**Rationale**: Ensures exact versions match technical specifications
**Impact**: More initial setup work, but precise version control

### 3. Simplified CSS Initially
**Decision**: Start with basic Tailwind setup, defer custom design tokens
**Rationale**: Avoid complexity until Shadcn/ui components are actually used
**Impact**: Will need to add CSS variables when implementing UI components

### 4. Documentation Structure
**Decision**: Create `documents/` folder for all reports and planning docs
**Rationale**: Clear separation between project docs and source code
**Impact**: Easy to track project progress and decisions over time
